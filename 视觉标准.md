# 品尚 AI 汇报 PPT · 浅色版视觉标准（Design Spec）

> 用途：本规范完整定义「品尚 AI 产品矩阵」浅色版汇报 PPT 的**配色、字体、布局、组件**标准。
> 后续可将本文件作为参数喂给 AI，按此标准生成同系列 PPT 模板，保证视觉一致。
> 设计基调：**浅色 · 深蓝科技风**。底色干净、蓝色强调、卡片化信息、流程/闭环图形化，杜绝长段落与表格堆叠。

---

## 0. 画布与基本规格

| 项 | 值 |
|---|---|
| 设计尺寸 | **1920 × 1080 px**（16:9，横版） |
| 缩放方式 | 固定画布 + `transform: scale()` 等比适配视口，黑色信箱式留边 |
| 安全边距（页面内边距） | 上下左右 **88px / 100px**（`padding: 88px 100px`） |
| 每页结构 | 顶部页眉（kick + 标题 + 核心句）→ 主体（卡片/流程/图）→ 底部高亮条（可选）→ 页脚 + 页码 |
| 单页信息密度 | 主体一般为 **3 列卡片 / 4 步流程 / 闭环图**，单页不超过一个主结构 |

## 1. 配色规范（Color Tokens）

所有颜色以 CSS 变量形式集中管理，禁止散落硬编码。

### 1.1 中性 / 底色

| Token | 色值 | 用途 |
|---|---|---|
| `--bg` | `#f4f6fa` | 页面主底色（极浅蓝灰） |
| `--panel` | `#ffffff` | 卡片 / 主面板背景 |
| `--panel-2` | `#eef2f8` | 次级面板、chip 背景、示例底 |
| `--panel-3` | `#e7edf6` | 三级面板 / 弱化标签底 |
| `--line` | `#e2e7f0` | 卡片描边、分隔线 |
| `--line-2` | `#d2dae7` | 较深描边、虚线分隔 |

页面背景叠加两处极淡蓝色径向光晕，增加科技氛围（右上 + 左下）：
```css
background:
  radial-gradient(1300px 760px at 82% -14%, #e8eefb 0%, transparent 55%),
  radial-gradient(1000px 640px at 4% 116%, #eef2fb 0%, transparent 52%),
  var(--bg);
```

### 1.2 文字色

| Token | 色值 | 用途 |
|---|---|---|
| `--ink` | `#111a2b` | 主标题、强调正文（近黑蓝） |
| `--ink-2` | `#37445e` | 正文、卡片描述 |
| `--muted` | `#65728c` | 次要说明、辅助文字 |
| `--faint` | `#9aa6ba` | 页码、超弱标签、占位 |

### 1.3 品牌强调色（深蓝科技）

| Token | 色值 | 用途 |
|---|---|---|
| `--accent` | `#2563eb` | 主强调蓝（标题高亮、图标线、序号） |
| `--accent-deep` | `#1d4ed8` | 深一档蓝（小标题、高亮条底、按钮） |
| `--accent-dim` | `#9db8ee` | 浅蓝（描边、箭头、连接线） |
| `--accent-wash` | `#eaf1fe` | 蓝色极浅底（图标底、强调卡片底、chip.accent） |

### 1.4 语义色（少量点缀）

| Token | 色值 | 用途 |
|---|---|---|
| `--pos` | `#0f9d6e` | 正向 / 成功（极少用） |
| `--neg` | `#dc5b4b` | 「过去 / 问题」标签、警示 |
| `--amber` | `#d9883a` | 提示（极少用） |
| 警示药丸底/边/字 | `#fdeee9` / `#f3c9bd` / `#b5503f` | 难点提示 pill |

### 1.5 用色原则
- 一页内**蓝色只作强调**，不大面积铺蓝；大色块仅用于「底部高亮条」与「价值总结卡」。
- 红/橙仅用于「过去 / 问题 / 难点」语义标记，每页最多出现一组。
- 卡片默认白底 + 浅描边；需要强调的卡片用 `--accent-wash` 底 + `--accent-dim` 边。

## 2. 字体规范（Typography）

### 2.1 字体族

```css
--ff-sans: 'Space Grotesk','Noto Sans SC', -apple-system, 'Segoe UI', sans-serif;
--ff-mono: 'JetBrains Mono','Noto Sans SC', ui-monospace, monospace;
```
- **西文/数字标题**：Space Grotesk（几何无衬线，科技感）
- **中文**：Noto Sans SC（思源黑体）
- **编号 / 标签 / 路径 / 页码**：JetBrains Mono（等宽，强化「技术/数据」感）

### 2.2 字号与字重层级（1920×1080 下）

| 角色 | 字号 | 字重 | 颜色 | 备注 |
|---|---|---|---|---|
| 封面主标题 H1 | **118px** | 700 | `--ink` / 高亮 `--accent` | `line-height:1.06`，`letter-spacing:-.03em` |
| 封面副标题 | 33px | 300 | `--muted`，强调 `--ink-2` | |
| 内页标题 `.s-title` | **54px** | 700 | `--ink`，高亮 `.a`=`--accent-deep` | `letter-spacing:-.02em` |
| 核心句 `.core` | 28px | 300 | `--ink-2`，强调 `b`=`--ink` 600 | 最宽 1620px |
| 卡片大标题 H3 | 28–36px | 700 | `--ink` | 视卡片密度而定 |
| 卡片正文 | 20–25px | 300 | `--ink-2` / `--muted` | |
| kick 眉标 `.kick` | 24px | 400 | `--muted`，段号 `.seg`=`--accent` 600 | mono 字体 |
| 标签 / chip | 24px | 500 | `--ink-2` | |
| 编号大数字 | 60–90px | 600 | `--accent-dim`/`--accent` | mono 字体 |
| 页脚 / 页码 | 17px | 400 | `--muted` / `--faint` | 属页面 chrome，可 <24px |
| 金句（结尾页） | 50px | 700 | `--ink`，高亮 `--accent` | |

### 2.3 排印细节
- 中文正文优先 `font-weight:300`（细），靠**字号与颜色**拉层级，而非全部加粗。
- 标题统一**负字距**（`letter-spacing:-.01em ~ -.03em`）收紧，更硬朗。
- mono 字体用于一切「序号、段落编号、英文小标、流程箭头、页码」——这是该风格的标志性手法。
- **正文最小 24px**（页脚/页码等 chrome 例外）；图内密集注释最低不小于 20px。

## 3. 布局规范（Layout）

### 3.1 页面骨架（统一模板）
```
┌────────────────────────────────────────────┐
│ kick 眉标（段号 · 副信息）                      │  ← font-mono, --muted
│ s-title 标题（含蓝色高亮词）                     │  ← 54px/700
│ core 核心句（一句话，可选）                       │  ← 28px/300
│                                              │
│ ▣ 主体区：3 列卡片 / 4 步流程 / 闭环图 / 对比     │  ← flex:1 撑满
│                                              │
│ ▮ 底部高亮条 sumbar（结论，可选）                │  ← 深蓝底白字
│ 页脚（段名 / 页码 — 总数）            页码（右下） │
└────────────────────────────────────────────┘
```

### 3.2 栅格与间距
- 主体多列统一用 **CSS Grid / Flex + `gap`**，禁止用 margin 拼间距。
- 常用列距/行距：卡片 `gap:22–28px`；高亮条 `margin-top:34px`；标题与核心句 `12–22px`。
- 圆角：大卡 `--r:18px`，小卡/药丸 `--r-sm:11px`，chip `10px`。
- 卡片内边距：大卡 `36px 34px`，紧凑卡 `26px 28px`。
- 描边统一 `1px solid var(--line)`；强调卡 `1px var(--accent-dim)`（必要时加 `box-shadow:0 0 0 1px`）。

### 3.3 页眉段标（导航锚点）
- `.kick` 用 `.seg` 显示所属章节（如 `01 现状说明` / `02 下一阶段重点方向` / `03 总结与展望`），蓝色加粗。
- 章节信息也写入页脚左下：`段名 / 当前页 — 总页数`，页码同时置右下。

### 3.4 底部高亮条 `.sumbar`（结论统一样式）
- 深蓝底 `--accent-deep` + 白字，左侧 mono 小标签（如「下一步 / 核心 / 判断」）+ 右侧结论句。
- 每个「现状/方向」类页面尽量以它收尾，形成统一节奏。

## 4. 组件库（Component Patterns）

| 组件 | 适用场景 | 结构要点 |
|---|---|---|
| **封面 cover** | 第 1 页 | 居中大标题 + 副标题；右侧 AI 节点线稿 SVG；左下 mono 元信息 |
| **三段式目录 agenda** | 第 2 页 | 每段：超大 mono 编号(90px) + 大标题 + 关键词 chips，整行卡片 |
| **三列信息卡 cards3** | 矩阵/方向总览 | 图标 chip + 角色小标(mono) + H3 + 一句定位 + 底部能力 chips |
| **问题卡 pcard** | 挑战/难点 | 编号(`问题一`) + H3 + 一句话 + 底部示例流程（`›` 连接） |
| **过去→未来 对比 ucard** | 升级方向 | 左标题(含 mono 序号) + 右「过去 box / → / 未来 box」，未来用蓝底 |
| **阶梯 ladder** | 递进逻辑 | 4 段，顶部进度条由短到长(末段实蓝) + 概念 + 行业示例 |
| **四步流程 steps4** | 操作路径 | 圆形 mono 序号 + H3 + 说明，卡间 `→` 连接符 |
| **前台/后台 split** | 简单 vs 复杂 | 左浅蓝「简单操作」流程列 + 右白底「复杂加工」+ 价值小格 |
| **两条路径 paths** | 主/补路径 | 左主路径(蓝边强调)编号链 + 右补充路径 + 警示 pill |
| **衍生应用 apps4** | 应用展开 | 4 张紧凑卡，mono 编号 + 名称 + 一句话 |
| **闭环 loop** | 园区业务闭环 | 左竖向 4 节点(圆号+连接线+关键词 chip) + 右深蓝价值卡 |
| **演进路径 evobar** | 市场趋势 | 横向 N 段 + `›` 箭头，末段蓝底高亮 |
| **结尾金句 punch** | 收尾页 | 3 段递进卡 + 大字金句(蓝色高亮关键词) |

### 4.1 图标规范
- 线性图标，`stroke=#2563eb`，`stroke-width:2.2`，无填充；放入 `.ico`（70px 圆角方，`--accent-wash` 底 + `--accent-dim` 边）。
- 禁止 emoji、禁止多色/立体图标、禁止图标堆砌（每卡最多 1 个）。

### 4.2 流程连接符
- 文内路径用 `›`（`.d` 浅蓝）；卡间用 `→`（`--accent-dim`，700）。统一蓝色系，不用杂色。

## 5. 动效规范（Motion）

- 仅做**入场**动画：元素 `.anim` 从 `translateY(20px)` + `opacity:0` 升起，`.6s cubic-bezier(.2,.7,.2,1)`，按 `nth-child` 递增 60ms 错峰。
- 动画**仅在激活页 `[data-deck-active]` 且 `prefers-reduced-motion:no-preference`** 时触发；基态即可见（保证打印 / PDF / 降级正常）。
- 不使用循环 / 装饰性持续动画。

---

## 6. 内容与版式纪律（务必遵守）

1. **图形化优先**：能用卡片/流程/闭环表达的，绝不写长段落；禁止表格堆叠。
2. **一句话原则**：核心句、卡片描述尽量一句话；示例用「A › B › C」短路径表达。
3. **一页一主结构**：每页只承载一个主图形（3 卡 / 4 步 / 闭环……），不混排。
4. **蓝色克制**：蓝色只作强调与结论块；正文保持中性黑蓝。
5. **统一收尾**：观点页用深蓝 `sumbar` 给结论，章节归属写进 kick 与页脚。
6. **留白充足**：宁少勿多，杜绝信息塞满；密度不够时用版式与留白解决，不靠加内容。

## 7. 给 AI 的最小生成参数（可直接复用）

```yaml
theme: light-deep-blue-tech
canvas: 1920x1080
fonts:
  display: Space Grotesk
  cjk: Noto Sans SC
  mono: JetBrains Mono
colors:
  bg: "#f4f6fa"
  panel: "#ffffff"
  ink: "#111a2b"
  ink2: "#37445e"
  muted: "#65728c"
  line: "#e2e7f0"
  accent: "#2563eb"
  accent_deep: "#1d4ed8"
  accent_dim: "#9db8ee"
  accent_wash: "#eaf1fe"
  warn: "#dc5b4b"
radius: { card: 18, small: 11 }
padding: 88px 100px
rules:
  - 每页：kick(段号) + s-title(含蓝高亮) + core(一句) + 主结构 + sumbar(结论)
  - 主结构择一：cards3 / steps4 / ladder / split / paths / loop / evobar
  - 编号·标签·箭头·页码一律 mono 字体
  - 蓝色仅作强调与结论块；正文 weight 300
  - 图标：线性单色 #2563eb，每卡≤1；禁 emoji / 表格 / 长段落
  - 入场动画仅激活页触发，基态可见
```
